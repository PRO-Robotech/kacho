# Project

**Project** — среднеуровневый контейнер внутри аккаунта ([Account](/api/account)). Это единица
изоляции рабочих нагрузок: именно к проекту привязываются ресурсы других сервисов —
VPC-сети и подсети, вычислительные инстансы, балансировщики. Проект группирует связанные ресурсы,
задаёт границу для квот и (главное) — **уровень выдачи прав**: роль можно привязать на конкретный
проект, ограничив доступ его содержимым.

Имя проекта уникально в пределах аккаунта (`(accountId, name)` — UNIQUE). Проект всегда
принадлежит ровно одному аккаунту.

## Методы

`ProjectService` — стандартный CRUD (`Create`/`Update`/`Delete` async → `Operation`;
`Get`/`List` sync) + журнал операций.

### Get — получить проект

<ApiOperation method="GET" endpoint="/iam/v1/projects/{project_id}">

Возвращает проект по идентификатору. Требует `v_get`.

<CodeBlock language="json">
  {dedent`
    {
      "id": "prj_7d3h5j9q2w4e6",
      "accountId": "acc_2k9f3qw8x7m2n",
      "name": "web-prod",
      "createdAt": "2026-06-24T10:05:00Z"
    }
  `}
</CodeBlock>

</ApiOperation>

### List — список проектов аккаунта

<ApiOperation method="GET" endpoint="/iam/v1/projects">

Возвращает проекты аккаунта, **видимые вызывающему** (default-deny через scope-filter, не 403):
handler оставляет только проекты, видимые через объединение `viewer ∪ v_list`. Обязателен
`accountId`; поддерживает `filter` по `name`.

<CodeBlock language="bash">
  {dedent`
    curl 'http://localhost:18080/iam/v1/projects?accountId=acc_2k9f3qw8x7m2n&pageSize=50' \\
      -H 'Authorization: Bearer <JWT>'
  `}
</CodeBlock>

</ApiOperation>

### Create — создать проект

<ApiOperation method="POST" endpoint="/iam/v1/projects" async>

Создаёт проект в аккаунте. Требует `editor` на аккаунте. Асинхронно (`response: Project`).

<CodeBlock language="bash">
  {dedent`
    curl -X POST 'http://localhost:18080/iam/v1/projects' \\
      -H 'Authorization: Bearer <JWT>' \\
      -H 'Content-Type: application/json' \\
      -d '{ "accountId": "acc_2k9f3qw8x7m2n", "name": "web-prod" }'
  `}
</CodeBlock>

</ApiOperation>

### Update — изменить проект

<ApiOperation method="PATCH" endpoint="/iam/v1/projects/{project_id}" async>

Меняет `name` / `description` / `labels` по `updateMask`. **`accountId` не меняется** через
Update (задан при Create). Требует `v_update`. Асинхронно.

<CodeBlock language="bash">
  {dedent`
    curl -X PATCH 'http://localhost:18080/iam/v1/projects/prj_7d3h5j9q2w4e6' \\
      -H 'Authorization: Bearer <JWT>' \\
      -H 'Content-Type: application/json' \\
      -d '{ "updateMask": "labels", "labels": { "tier": "frontend" } }'
  `}
</CodeBlock>

</ApiOperation>

### Delete — удалить проект

<ApiOperation method="DELETE" endpoint="/iam/v1/projects/{project_id}" async>

Удаляет строку проекта. Требует `v_delete`. Асинхронно.

</ApiOperation>

:::warning Cross-service ссылки на проект
Ресурсы других сервисов (Network, Instance, ...) ссылаются на проект по `projectId` без
cross-service FK (database-per-service). Consumer-сервисы обязаны грациозно переживать
dangling-ссылку на удалённый проект — жёстких cross-service cascade нет.
:::

### ListOperations — операции над проектом

<ApiOperation method="GET" endpoint="/iam/v1/projects/{project_id}/operations">

Возвращает операции над проектом. Sync, курсорная пагинация. Требует `v_list`.

</ApiOperation>
