# User

**User** — человек-субъект платформы: сотрудник, которому выдаётся доступ. В модели Kachō User —
это **проекция личности** из провайдера аутентификации (Ory Kratos) в пределах одного аккаунта.
Один и тот же человек (одна Kratos-identity) может иметь несколько User-строк — по одной на
каждый аккаунт, куда он приглашён. Пользователь — это то, к чему привязываются роли; сам вход
(пароль / passkey) живёт в Kratos, а IAM хранит лишь снимок личности и её статус.

Пользователи заводятся не через `Create`, а через **приглашение** (`Invite`) по email: backend
создаёт `PENDING`-строку, которая становится `ACTIVE` при первом входе через Kratos с совпадением
email.

## Методы

`UserService` — чтение (`Get`/`List`), приглашение (`Invite`), обновление меток (`Update`),
удаление (`Delete`) и журнал операций. Классического `Create` нет — пользователи появляются
только через `Invite`.

### Get — получить пользователя

<ApiOperation method="GET" endpoint="/iam/v1/users/{user_id}">

Возвращает пользователя по идентификатору. Требует `v_get`.

<CodeBlock language="json">
  {dedent`
    {
      "id": "usr_5m1p7rt4b9k3d",
      "externalId": "ory:kratos:9f2c…",
      "email": "dev@acme.example",
      "displayName": "Dev One",
      "accountId": "acc_2k9f3qw8x7m2n",
      "inviteStatus": "ACTIVE",
      "createdAt": "2026-06-24T10:00:00Z"
    }
  `}
</CodeBlock>

</ApiOperation>

### List — список пользователей аккаунта

<ApiOperation method="GET" endpoint="/iam/v1/users">

Возвращает пользователей аккаунта, **видимых вызывающему** (default-deny через scope-filter,
не 403): handler оставляет только пользователей аккаунтов, где вызывающий — участник. Обычному
пользователю `accountId` обязателен; поддерживает `filter` по `email` / `externalId` /
`inviteStatus`.

<CodeBlock language="bash">
  {dedent`
    curl 'http://localhost:18080/iam/v1/users?accountId=acc_2k9f3qw8x7m2n&pageSize=50' \\
      -H 'Authorization: Bearer <JWT>'
  `}
</CodeBlock>

:::note Пользователь всегда видит себя
Вызывающий всегда присутствует в результате `List` своего аккаунта (self-floor) даже без явной
роли — иначе он не смог бы найти собственную запись.
:::

</ApiOperation>

### Invite — пригласить пользователя

<ApiOperation method="POST" endpoint="/iam/v1/users:invite" async>

Приглашает пользователя по email: создаётся `PENDING`-строка. Идемпотентно — повторное
приглашение уже приглашённого возвращает `Operation` без дублирования. Опционально — сразу
выдать роль на проекте (`projectId` + `roleId`), тогда AccessBinding создаётся атомарно.
Требует `editor` на аккаунте. Асинхронно (`response: User`).

<CodeBlock language="bash">
  {dedent`
    curl -X POST 'http://localhost:18080/iam/v1/users:invite' \\
      -H 'Authorization: Bearer <JWT>' \\
      -H 'Content-Type: application/json' \\
      -d '{
            "accountId": "acc_2k9f3qw8x7m2n",
            "email": "dev@acme.example",
            "displayName": "Dev One"
          }'
  `}
</CodeBlock>

`Operation.metadata` (`InviteUserMetadata`) несёт `magicLinkUrl` — recovery/magic-link Kratos,
который админ передаёт приглашённому вручную (автоотправка email не интегрирована). Для
повторного приглашения уже активного пользователя ссылка пустая.

</ApiOperation>

### Update — изменить метки пользователя

<ApiOperation method="PATCH" endpoint="/iam/v1/users/{user_id}" async>

Единственное mutable-поле — `labels` (делает пользователя label-selectable наравне с
account/project). Identity-поля (`externalId` и прочие, проецируемые из IdP) hard-immutable: их
наличие в `updateMask` → `INVALID_ARGUMENT` (`"<field> is immutable after User.Create"`).
Требует `v_update`. Асинхронно.

<CodeBlock language="bash">
  {dedent`
    curl -X PATCH 'http://localhost:18080/iam/v1/users/usr_5m1p7rt4b9k3d' \\
      -H 'Authorization: Bearer <JWT>' \\
      -H 'Content-Type: application/json' \\
      -d '{ "updateMask": "labels", "labels": { "team": "platform" } }'
  `}
</CodeBlock>

</ApiOperation>

### Delete — удалить пользователя

<ApiOperation method="DELETE" endpoint="/iam/v1/users/{user_id}" async>

Удаляет пользователя. Требует `v_delete`. Асинхронно.

</ApiOperation>

### ListOperations — операции над пользователем

<ApiOperation method="GET" endpoint="/iam/v1/users/{user_id}/operations">

Возвращает операции над пользователем. Sync, курсорная пагинация. Требует `v_list`.

</ApiOperation>

<table>
  <thead><tr><th>Ситуация</th><th>Код</th><th>Поведение</th></tr></thead>
  <tbody>
    <tr><td>Get/Update/Delete несуществующего пользователя</td><td><code>NOT_FOUND</code></td><td><code>"User &lt;id&gt; not found"</code></td></tr>

    <tr><td>List без <code>accountId</code> (обычный пользователь)</td><td><code>INVALID_ARGUMENT</code></td><td>Scope-фильтр требует аккаунт</td></tr>
  </tbody>
</table>

:::tip Активация при первом входе
Приглашённый пользователь становится `ACTIVE` автоматически при первом успешном входе через
Kratos: hook сопоставляет email и заполняет `externalId`. Механика — на странице
[Идентичность и токены](/architecture/identity).
:::
