# Конфигурация

Kachō IAM конфигурируется через переменные окружения с префиксом **`KACHO_IAM_`** (плюс
опциональный YAML-файл через `KACHO_IAM_CONFIG_PATH`). Эта страница перечисляет ключевые группы
настроек: подключение к БД, OpenFGA, провайдеры идентичности (Kratos / Hydra), listener'ы и mTLS,
режим аутентификации. Секреты (пароли, токены) подаются из окружения / секрет-хранилища, не
хардкодятся.

## Listener'ы и mTLS

Сервис поднимает публичный (`:9090`), internal (`:9091`), hooks (`:9092`) и metrics (`:9095`)
listener'ы. Каждый может требовать mTLS.

<table>
  <thead><tr><th>Переменная</th><th>Назначение</th></tr></thead>
  <tbody>
    <tr><td><code>KACHO_IAM_GRPC_PORT</code> / <code>KACHO_IAM_INTERNAL_PORT</code></td><td>Порты public / internal listener'ов</td></tr>
    <tr><td><code>KACHO_IAM_PUBLIC_SERVER_MTLS_…</code></td><td>mTLS публичного listener'а (cert / key / clientCA / enable)</td></tr>
    <tr><td><code>KACHO_IAM_INTERNAL_SERVER_MTLS_…</code></td><td>mTLS internal listener'а (<code>InternalIAMService</code>)</td></tr>
    <tr><td><code>KACHO_IAM_HOOKS_SERVER_MTLS_…</code></td><td>mTLS hooks-listener'а (Kratos identity-хуки)</td></tr>
    <tr><td><code>KACHO_IAM_METRICS_SERVER_MTLS_…</code></td><td>mTLS metrics-listener'а (Prometheus)</td></tr>
    <tr><td><code>KACHO_IAM_METRICS_ENDPOINT</code></td><td>Адрес Prometheus <code>/metrics</code> (cluster-internal, по умолчанию <code>:9095</code>)</td></tr>
    <tr><td><code>KACHO_IAM_GATEWAY_INTERNAL_…</code></td><td>Адрес и клиентский mTLS до internal-mux api-gateway (subject-change)</td></tr>
  </tbody>
</table>

## Режим аутентификации

<table>
  <thead><tr><th>Переменная</th><th>Назначение</th></tr></thead>
  <tbody>
    <tr><td><code>KACHO_IAM_AUTHN__MODE</code></td><td>Режим аутентификации (dev-фикстуры vs production-strict)</td></tr>

  </tbody>
</table>

:::warning Production-режим — fail-closed
В production-профиле сервис обязывает mTLS на публичном (`:9090`) и internal (`:9091`)
listener'ах и сконфигурированный OpenFGA. Без них он **откажется стартовать** («refusing to start
with insecure :9090/:9091») — никакого молчаливого insecure-downgrade. Анонимный доступ отклонён
на всех портах. См. [Особенности дизайна](/advanced/design-decisions).
:::

## Пример (локальный dev)

<CodeBlock language="bash">
  {dedent`
    export KACHO_IAM_DB_HOST=localhost
    export KACHO_IAM_DB_PORT=5432
    export KACHO_IAM_DB_USER=kacho
    export KACHO_IAM_DB_PASSWORD=secret
    export KACHO_IAM_DB_NAME=kacho_iam
    export KACHO_IAM_DB_SSLMODE=disable
    export KACHO_IAM_OPENFGA_ENDPOINT=localhost:8080

    bin/kacho-migrator up
    bin/kacho-iam serve
  `}
</CodeBlock>
