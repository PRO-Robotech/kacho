# Конфигурация

Kachō API Gateway конфигурируется через переменные окружения. Большинство — с префиксом
**`KACHO_API_GATEWAY_`**; ключи AuthN-инфраструктуры (Hydra / JWKS / DPoP) — с префиксом
`KACHO_` (общие для платформы). Эта страница перечисляет ключевые группы и даёт пример.

:::warning Secure-by-default
При `KACHO_APP_ENV=production` гейтвей **откажется стартовать**, если authz выключен
(`AUTHZ_ENABLED=false`), включён fail-open (`AUTHZ_FAIL_OPEN=true`), AuthN не в production-режиме
или internal gRPC-listener не защищён mTLS. Пустой/неустановленный `KACHO_APP_ENV` (дефолт)
пропускает гейт — dev-режим для локальных стендов.
:::

## mTLS к backend'ам (per-edge)

## Пример YAML

<CodeBlock language="yaml" title="api-gateway.env.yaml">
  {dedent`
    KACHO_APP_ENV: production

    KACHO_API_GATEWAY_LISTEN_ADDR: ":8080"
    KACHO_API_GATEWAY_TLS_LISTEN_ADDR: ":443"
    KACHO_API_GATEWAY_TLS_CERT_FILE: /tls/tls.crt
    KACHO_API_GATEWAY_TLS_KEY_FILE: /tls/tls.key

    # AuthN — production Hydra + DPoP
    KACHO_API_GATEWAY_AUTHN_MODE: production-strict
    KACHO_API_GATEWAY_AUTHN_ENABLE_DPOP: "true"
    KACHO_HYDRA_ISSUER: https://auth.kacho.cloud/
    KACHO_HYDRA_JWKS_URL: https://auth.kacho.cloud/.well-known/jwks.json

    # AuthZ — включён, fail-closed
    KACHO_API_GATEWAY_AUTHZ_ENABLED: "true"
    KACHO_API_GATEWAY_AUTHZ_FAIL_OPEN: "false"
    KACHO_API_GATEWAY_IAM_AUTHORIZE_URL: iam.kacho.svc.cluster.local:9091

    # mTLS к backend'ам
    KACHO_API_GATEWAY_MTLS_CLIENT_CERT_FILE: /mtls/client.crt
    KACHO_API_GATEWAY_MTLS_CLIENT_KEY_FILE: /mtls/client.key
    KACHO_API_GATEWAY_MTLS_CA_FILE: /mtls/ca.crt
    KACHO_API_GATEWAY_MTLS_IAM_ENABLE: "true"
    KACHO_API_GATEWAY_MTLS_VPC_ENABLE: "true"

    # Internal listener mTLS (push-инвалидация от iam)
    KACHO_API_GATEWAY_INTERNAL_GRPC_MTLS_ENABLE: "true"
    KACHO_API_GATEWAY_INTERNAL_GRPC_ALLOWED_SPIFFE: spiffe://kacho.cloud/ns/kacho/sa/kacho-iam
  `}
</CodeBlock>

:::tip Локальный dev
Для локального запуска достаточно `KACHO_API_GATEWAY_AUTHN_MODE=dev` (остальное берёт дефолты):
`KACHO_API_GATEWAY_AUTHN_MODE=dev bin/api-gateway`. Режим `dev` поднимает гейтвей без mTLS и без
жёстких authz-гейтов. Сборка — в [Развёртывании](/install/deploy).
:::
